# Testing Workshop

Учебный проект для воркшопа по теме `Юнит-тесты: инъекции и разрыв зависимостей`.

## Что внутри

- `RewardIssuingService` показывает нормальный DI через конструктор и подходит для unit-тестов с mock/stub.
- `LegacyCertificateService` специально завязан на время, UUID и файловую систему. Он нужен для практики по разрыву зависимостей и рефакторингу.
- `RewardController` дает простые HTTP-ручки, если захочется быстро показать приложение вживую.

## Практика для студентов

1. Написать unit-тесты для `RewardIssuingService`.
2. Покрыть минимум такие сценарии:
   - неактивный пользователь;
   - слишком маленькая покупка;
   - fraud check отклоняет операцию;
   - для сегмента нет кампании;
   - награда успешно выдается;
   - сообщение отправляется только для большой награды.
3. Разобрать, какие зависимости у `RewardIssuingService` внешние и почему именно их стоит подменять в unit-тестах.
4. Отрефакторить `LegacyCertificateService`, чтобы его стало удобно тестировать.
5. После рефакторинга добавить unit-тесты на генерацию сертификата.

## Идеи для рефакторинга `LegacyCertificateService`

- внедрить `Clock` вместо прямого вызова `LocalDate.now()`;
- выделить генератор кода сертификата;
- вынести файловую систему за интерфейс;
- сделать побочные эффекты отдельной зависимостью, а бизнес-решение оставить внутри сервиса.

## Запуск

```bash
cd testing-workshop
./mvnw spring-boot:run
```

## Примеры запросов

Выдать награду:

```bash
curl -X POST http://localhost:8080/api/rewards/issue \
  -H "Content-Type: application/json" \
  -d '{"customerId":3,"purchaseAmount":8000}'
```

Сгенерировать сертификат:

```bash
curl -X POST http://localhost:8080/api/certificates \
  -H "Content-Type: application/json" \
  -d '{"studentEmail":"student@example.com","solvedTasks":4}'
```
